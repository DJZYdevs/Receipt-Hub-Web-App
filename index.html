Archive & storage

      
    

    

      
In this prototype, receipts are stored locally in your browser (IndexedDB-style memory) so you can test the flow. In a production build, each verified receipt would sync automatically to a real backend so nothing lives only on one device.

      
Recommended production setup

      

        ☁️
        
Cloud object storage + databaseOriginal photos go to a bucket (e.g. Google Cloud Storage / S3 / Cloudflare R2), organised by folder/year/month. Extracted fields go into a proper database (e.g. Postgres via Supabase or Firebase) so they're searchable and reportable.

      

      

        🔗
        
Connect your own Google DriveSimplest for a solo user — receipts save into folders like Drive/Receipts/Work/2026-07, with the sheet data mirrored to a Google Sheet for easy export to your accountant.

      

      

        🔒
        
Encrypted local-first, optional syncWorks offline on your phone; syncs to cloud only when connected. Best if you want data to stay private by default.

      

      
What gets archived per receipt

      
Original image(s), extracted fields, your verified/edited values, timestamp of verification, and folder/category — so every entry is fully auditable later.


      
OCR engine (text extraction)

      
This prototype uses OCR.space — a real, free OCR API (25,000 requests/month free tier). It's currently running on the shared public demo key, which is rate-limited and shared with other developers testing the same API, so you may occasionally see a "rate limit" error if a lot of people are testing it at once.

      

        
        
      

      
Get your own free key in 30 seconds at ocr.space/ocrapi (no credit card) and paste it above — it's saved in this browser only and unlocks much higher limits.
